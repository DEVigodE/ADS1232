╔════════════════════════════════════════════════════════════════════╗

╠════════════════════════════════════════════════════════════════════╣

║ (Array Fixo)         │          │         │         │ (definida)     ║
╠════════════════════════════════════════════════════════════════════╣
║ Fila Encadeada       │   O(1)   │  O(1)   │ Dinâmica│ Ilimitada      ║
║ (Linked List)        │          │         │         │ (só memória)   ║
╚════════════════════════════════════════════════════════════════════╝

VANTAGENS E DESVANTAGENS:
------------------------------

🏥 FILA ORIGINAL (ArrayList):
  ✓ Fácil implementação
  ✓ Suporte a prioridade
  ✗ Remoção custosa O(n) - desloca elementos
  ✗ Ineficiente para muitas operações de dequeue

🔄 FILA CIRCULAR (Array Fixo):
  ✓ Inserção e remoção O(1)
  ✓ Reutiliza espaço do array
  ✓ Baixo overhead de memória
  ✗ Capacidade limitada
  ✗ Precisa dimensionamento correto

🔗 FILA ENCADEADA (Linked List):
  ✓ Inserção e remoção O(1)
  ✓ Capacidade dinâmica ilimitada
  ✓ Flexível para crescimento
  ✗ Overhead de ponteiros
  ✗ Localidade de memória pior

📊 RECOMENDAÇÃO DE USO:
-------------------------
• Fila Circular: Para sistemas com capacidade conhecida e muitas operações
• Fila Encadeada: Para sistemas com demanda variável e crescimento dinâmico
• Fila Original: Para prototipagem rápida ou quando prioridade é necessária

🔍 DISCUSSÃO DETALHADA:
-------------------------

1️⃣ FILA DE PRIORIDADE com ARRAYLIST:
   ✅ INSERÇÃO O(1): Adiciona sempre no final da lista apropriada
   ❌ REMOÇÃO O(n): remove(0) precisa deslocar todos os elementos
   ✅ BUSCA O(1): Acesso direto ao primeiro elemento
   📝 Mantém 3 listas separadas (urgente, idoso, normal)

2️⃣ FILA DE PRIORIDADE ENCADEADA:
   ❌ INSERÇÃO O(n): Precisa encontrar posição correta na ordem
   ✅ REMOÇÃO O(1): Remove sempre do início, sem deslocamentos
   ✅ BUSCA O(1): Acesso direto ao primeiro nó
   📝 Mantém ordem única por prioridade usando ponteiros

🚀 QUANDO USAR CADA ESTRUTURA:
-----------------------------------

🏥 FILA DE PRIORIDADE ARRAYLIST:
   • Ideal quando: Muitas inserções, poucas remoções
   • Cenário: Cadastro em lote, atendimento esporádico
   • Vantagem: Inserção muito rápida O(1)
   • Desvantagem: Remoção custosa O(n)

🔗 FILA DE PRIORIDADE ENCADEADA:
   • Ideal quando: Muitas remoções, inserções controladas
   • Cenário: Atendimento contínuo, chegadas esparsas
   • Vantagem: Remoção muito rápida O(1)
   • Desvantagem: Inserção pode ser custosa O(n)

⚖️ FILA ENCADEADA vs ARRAY - VANTAGENS:
---------------------------------------------

🔗 LISTA ENCADEADA É MELHOR QUANDO:
   1. Frequência alta de remoções (chamadas de pacientes)
   2. Tamanho da fila varia muito dinamicamente
   3. Não há limite pré-definido de capacidade
   4. Prioridade dos elementos muda frequentemente
   5. Sistema tem pouca memória disponível (sem desperdício)

📊 ARRAY É MELHOR QUANDO:
   1. Acesso frequente por índice
   2. Capacidade conhecida e estável
   3. Cache locality é importante
   4. Overhead de ponteiros deve ser minimizado
   5. Muitas inserções em batch

💡 RECOMENDAÇÃO PRÁTICA:
-------------------------
Para sistemas hospitalares reais:
• Use ENCADEADA se atendimento é contínuo (pronto-socorro)
• Use ARRAYLIST se chegadas são em lote (agendamentos)
• Considere HEAP para grandes volumes com prioridades complexas
